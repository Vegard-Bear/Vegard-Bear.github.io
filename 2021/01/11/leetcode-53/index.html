53. 最大子序和

题目描述

给定一个整数数组 nums ，找到一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

示例:

输入: [-2,1,-3,4,-1,2,1,-5,4]
输出: 6
解释: 连续子数组 [4,-1,2,1] 的和最大，为 6。
  • 进阶:

如果你已经实现复杂度为 O((n)) 的解法，尝试使用更为精妙的分治法求解。

解题思路

这道题与 最长递增子序列 非常相似，这里我们依旧是采用动态规划算法解决。

定义dp数组

dp[i] 表示以 nums[i] 结尾的最大子数组和

确定 base case

那么根据这个定义。我们就可以确定 base case，就是

dp[i] = 0

因为在初始状态下，以nums[i]结尾的最大子数组和就是其自身。
然后就可以列我们的状态转移方程了。

推导状态转移方程

依旧是采用数学归纳法来寻找状态转移关系：假设我们已经计算出了dp[i-1]，如何推导出dp[i]呢？

dp[i]其实只有两种选择，要么就是与前面的相邻子数组连接，形成一个更大的子数组，要么不与前面的子数组比较，自成一派，自己作为一个子数组。

而对于这两种情况，我们只需要其中和最大的那一个，也就是：

dp[i] = max (( dp[i] , dp[i-1] + nums[i] ))

至此，我们列出状态转移方程，就可以直接写代码了。

题解

func maxSubArray(nums []int) int {
    // 定义dp动态数组, dp表示以nums[i]结尾的最大子数组和
    dp := make([]int, len(nums))

    // 定义base case, 第一个数前面没有数，所以
    // 以第一个数结尾的最大子数组和就是其本身
    dp[0] = nums[0]

    // 开始计算dp数组
    for i := 1; i < len(nums); i++{
        // 列出状态转移方程
        // 以nums[i]结尾的最大数组和 要么是其自身nums[i]，
        // 要么是自身nums[i]加上以前一个数nums[i-1]结尾的最大子数组拼上当前数而成的数组和
        dp[i] = max(nums[i], dp[i-1] + nums[i])
    }
     // res用于返回最后结果
    res := math.MinInt32
    // 遍历dp数组，找到最大的那一个就是答案
    for i := 0; i < len(dp); i++ {
        res = max(res, dp[i])
    }
    return res
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

不难发现，以上解法的时间复杂度和空间复杂度都是O((n)), 考虑到 dp[i]仅仅和dp[i-1]有关，那么就可以进行状态压缩，进一步降低空间复杂度

func maxSubArray(nums []int) int {
    // 定义base case，注意这里只定义了dp_0，而不是整个dp数组
    dp_0 := nums[0]
    res := dp_0

    // 开始计算dp数组
    for i := 1; i < len(nums); i++ {
        // 列出状态转移方程
        dp_1 := max(nums[i], dp_0 + nums[i])
        // 注意这里需要更新前一个状态
        dp_0 = dp_1
        // 直接计算当前最大和
        res = max(res, dp_1)
    }
    return res
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

复杂度分析

时间复杂度：O((n))

其中 n 为数组 nums 的长度。

空间复杂度：O((C))

不再需要额外定义一个dp数组了，只需要dp_0和dp_1两个变量，所以此算法的空间复杂度为常数。

反思

动态规划算法可通过状态压缩进一步降低空间复杂度。

注意点

本题与 最长递增子序列的不同在于，递增子序列是可以不连续的，因此出现二层循环，扫描当前元素之前的任何一个序列，取最大值，而本题要求连续子数组，这样就不能扫描之前的所有元素了，只能扫描与之相邻的元素，也就是前一个元素。
