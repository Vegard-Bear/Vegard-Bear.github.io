516. 最长回文子序列

题目描述

给定一个字符串 s ，找到其中最长的回文子序列，并返回该序列的长度。可以假设 s 的最大长度为 1000 。

示例 1:

输入:

"bbbab"

输出:

4

一个可能的最长回文子序列为 “bbbb”。

示例 2:

输入:

"cbbd"

输出:

2

一个可能的最长回文子序列为 “bb”。

提示：

  • 1 <= s.length <= 1000
  • s 只包含小写英文字母

解题思路

这道题和最长公共子序列以及编辑距离套用的都是二维的dp数组模版

动态规划三步走

定义dp数组

这道题和最长公共子序列以及编辑距离套用的都是二维的dp数组模版，但是不同之处在于这里只涉及一个字符串/子序列，所以属于二维dp数组的另一种小类。

dp[i][j] 表示子串s[i…j]的最长回文子序列长度

确定 base case

根据dp数组的定义。我们就可以确定 base case，即

当 i 等于 j 时, dp[i][j] = 1

这是因为i等于j时，dp[i][j]表示单个字符，而单个字符一定是回文序列，且长度为1，也就是说二维dp数组的对角线上都是1

推导状态转移方程

依旧是采用数学归纳法来寻找状态转移关系：假设我们已经计算出了dp[i-1][j-1]，dp[i-1][j]以及dp[i][j-1],如何推导出dp[i][j]呢？

对于dp[i][j]有两种选择：

  1. 如果s[i]等于s[j]，也就是首尾两个字符相等，那么直接拼上中间的最长回文序列，形成一个更长的回文子序列，此时子序列的长度+2。
  2. 如果s[i]不等于s[j]，也就是首尾两个字符不相等，那么就将这两个字符分别加入中间的最长回文子序列，取其中更长的那个即可。
             dp[i+1][j-1]+2                   当s[i]==s[j]时
dp[i][j] = 
             max(dp[i+1][j], dp[i][j-1])      当s[i]!=s[j]时

dp数组遍历方式

最后还剩下一个问题，就是dp数组的遍历方式，注意这里其实是有两种遍历方式的，我们取其中一种即可，至于具体怎么选择，可参考 子序列问题通用思路|最长回文子序列

至此，我们就可以直接写代码了。

题解

func longestPalindromeSubseq(s string) int {
    // 初始化二维dp数组
    // dp[i][j]表示子串s[i...j]的最长回文子序列长度
    n := len(s)
    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, n)
    }
    // 确定base case
    // 注意对角线上是1，这是因为dp[i][i]表示单个字符，而单个字符一定是回文序列，且长度为1
    for i := 0; i < n ; i++ {
        dp[i][i] = 1
    }
    // 推导状态转移方程，注意这里dp数组的遍历方式
    for i := n - 2 ; i >= 0 ; i-- {
        for j := i + 1 ; j < n ; j++ {
            // 如果首尾两个字符相等，那么直接拼上中间的最长回文序列
            if s[i] == s[j] {
                dp[i][j] = dp[i+1][j-1] +2
            } else {
                // 如果首尾两个字符不相等，那么就将这两个字符分别加入中间的最长回文子序列，取其中更长的那个即可
                dp[i][j] = max(dp[i+1][j], dp[i][j-1])
            }
        }
    }
    // 最后返回整个字符串s的最长回文子序列
    return dp[0][n-1]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

复杂度分析

时间复杂度：O((n^2))

n 为字符串 s 的长度

空间复杂度：O((n^2))

我们需要大小为 O((n^2)) 的 dp 数组来记录状态值。

反思

1. 这道题和最长公共子序列以及编辑距离套用的都是二维的dp数组模版，但是不同之处在于这里只涉及一个字符串/子序列，所以属于二维dp数组的另一种小类，是一种隐藏的二维动态规划。
2. 子序列和字串更难，因为可能是不连续的，几乎肯定是用动态规划做的，这道题和最长公共子序列以及编辑距离还有区别，它的难点在于推导动态规划的时候的数组遍历顺序，需要逆着遍历，具体原因可以参考子序列问题通用思路|最长回文子序列

几个注意点

  1. 回文子序列也是用的二维动态规划
  2. 这里的二维动态规划是不需要把i=0和j=0这两行单独空出来的（大多数情况下需要）
  3. 这里n的遍历顺序是逆着来的
  4. 注意j永远大于i
